<Admonition type="warning">
The `system.loadAddon()` method is **only available in the Main script**. Native addons are **engine-agnostic** and do not require linking against Duktape.
</Admonition>

Novadesk allows you to extend its functionality by loading native C++ DLLs (addons). This is useful for performance-critical tasks or accessing system features not available in JavaScript.

## Methods
<Method
  signature="system.loadAddon(path)"
  description="Loads a native addon DLL into the Novadesk environment."
>
  ### Parameters
  <Admonition type="tip">
  * **`path`** (`string`): The relative or absolute path to the `.dll` file. 
      Relative paths are resolved starting from the directory where the current script is located.
  </Admonition>
  
  <ReturnValue type="object|null">
    Returns a JavaScript object containing the API exported by the addon, or `null` if the addon fails to load or initialize.
  </ReturnValue>
</Method>

<Method
  signature="system.unloadAddon(path)"
  description="Unloads a previously loaded native addon. This disposes of the native resources and calls the addon's optional cleanup code."
>
  ### Parameters
  * **`path`** (`string`): The relative or absolute path to the `.dll` file. This **must match exactly** the path provided to `loadAddon()`.
  
  <ReturnValue type="boolean">
    Returns `true` if the addon was successfully found and unloaded; `false` otherwise.
  </ReturnValue>
</Method>

## Examples

<CodeBlock language="javascript">
  {`// Load a local addon
const myAddon = system.loadAddon("./utils_addon.dll");

if (myAddon) {
    console.log("Addon version: " + myAddon.version);
    
    // Use functions provided by the addon
    const result = myAddon.calculate(10, 20);
    console.log("Result: " + result);
    
    // Unload when finished (voluntary)
    system.unloadAddon("./utils_addon.dll");
}`}
</CodeBlock>
