<Admonition type="warning">
The `system` object is **only available in the Main script**. UI scripts should communicate with the main script via [IPC](/api/widget-api/widget-methods/#inter-process-communication-ipc) if they need system data.
</Admonition>

## Methods

<Method signature="system.execute(target, [parameters], [workingDir], [show])"
  description="Launches an external application, opens a file, or opens a URL in the default browser.">
  <MethodParam name="target" type="string">
    Path to the executable, file, or a URL.
  </MethodParam>
  <MethodParam name="parameters" type="string">
    Command-line arguments passed to the target. Only used when `target` is an executable.
  </MethodParam>
  <MethodParam name="workingDir" type="string">
    Directory in which the command should be executed.
  </MethodParam>
  <MethodParam name="show" type="string | number">
    **Default**: `"normal"`

    Controls how the window is displayed.

    **Valid values (string)**:
    - **`"hide"`**
    - **`"normal"`**
    - **`"minimized"`**
    - **`"maximized"`**
  </MethodParam>
  ### Return
  <ReturnValue type="boolean">
    Returns `true` if the command was successfully initiated. Returns `false` if the operation fails.
  </ReturnValue>
</Method>

## Examples

#### Launch an Application
<CodeBlock language="javascript">
  {`system.execute("notepad.exe");`}
</CodeBlock>

#### Open a URL
<CodeBlock language="javascript">
  {`system.execute("https://www.google.com");`}
</CodeBlock>

#### Run a Command with Parameters
<CodeBlock language="javascript">
  {`system.execute("cmd.exe", "/c dir", "C:\\", "normal");`}
</CodeBlock>
