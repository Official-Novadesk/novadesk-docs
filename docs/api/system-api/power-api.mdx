<Admonition type="warning">
The `system` object is **only available in the Main script**. UI scripts should communicate with the main script via [IPC](/api/widget-api/widget-methods/#inter-process-communication-ipc) if they need system data.
</Admonition>

The Power API provides information about the system's power source, battery status, and CPU frequency.

---

<Method
  signature="system.getPowerStatus()"
  description="Retrieves current power and battery information."
>
  <ReturnValue type="object | null">
    Returns an object with the following properties, or `null` if the information could not be retrieved.
    
    ### `acline` (number)
    `1` if plugged in (AC), `0` otherwise (Battery).
    
    ### `status` (number)
    Mapped battery status:
    * `0`: No battery
    * `1`: Charging
    * `2`: Critical
    * `3`: Low
    * `4`: High/Full
    
    ### `status2` (number)
    Raw `BatteryFlag` from Windows.
    
    ### `lifetime` (number)
    Estimated battery life remaining in seconds. Returns `-1` if unknown.
    
    ### `percent` (number)
    Battery charge percentage (0-100).
    
    ### `mhz` (number)
    Current CPU frequency in MHz.
    
    ### `hz` (number)
    Current CPU frequency in Hz (MHz * 1,000,000).
  </ReturnValue>
</Method>

## Examples

<CodeBlock language="javascript">
  {`var power = system.getPowerStatus();

if (power) {
    console.log("Source: " + (power.acline ? "AC Power" : "Battery"));
    console.log("Battery: " + power.percent + "%");
    console.log("CPU Frequency: " + power.mhz + " MHz");
    
    if (power.status === 1) {
        console.log("Status: Charging");
    }
}`}
</CodeBlock>
